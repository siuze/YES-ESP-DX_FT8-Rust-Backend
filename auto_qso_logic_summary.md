# FT8 持续自动通联模式 (Mode 3 / Auto Hunter) 运行逻辑全景指南

在 `YES-ESP-DX_FT8-Rust-Backend` 系统中，自动通联功能由一个严密的时钟调度器驱动，结合高优先级的策略大脑和底层的防死锁跳频机制，实现了一套极高容错率的全自动 FT8 握手机制。

---

## 1. 核心时序与任务调度 (`tx_control.rs`)

整个后端的状态机与时钟紧密绑定，以 100ms 的 Tick 轮询当前时间，并在特定的秒数边缘触发动作：

- **发射动作点 (`0s, 15s, 30s, 45s`)**：
  在进入新的 FT8 时隙开端的 500ms 内，系统会检查队列中是否有准备好的待发射消息（`pending_msg`），并且检查当前时隙的奇偶性（Even/Odd）是否与当前任务所期望的 `tx_window_even` 一致。如果吻合，系统下发 UDP 发射指令，并将当前任务的 `repeat_count` 加 1。
  *(容错机制：即使 UDP 下发失败或无网络，系统也必定加 1 以防止死锁)*

- **策略决策点 (`14s, 29s, 44s, 59s`)**：
  在对方可能刚刚发完消息的接收时隙末端（解码已出结果），如果系统当前队列为空（`is_idle` 为 `true`），则系统会呼叫大脑 `qso_auto_hunter.rs` 重新评估局势，并分配下一个 15 秒周期的任务。

---

## 2. 战术大脑：打擂台与游击战 (`qso_auto_hunter.rs`)

在策略决策点，系统会按照极高纪律性的优先级，判定是自己打擂台（CQ）还是四处游击（Chase）。

### 优先级 A：强制 / 紧急自动 CQ (Auto CQ)
系统会在以下几种情况下终止“游击”，强行自己做主站发 CQ：
1. **多次碰壁**：连续主动追踪呼叫别人 `>= 2` 次（`consecutive_failures >= 2`），但都没有得到回复。
2. **长时间冷场**：已经超过 10 分钟没有成功通联，或是距离上一次发 CQ 已经过了 3 分钟（防止一直做小透明）。

**CQ 的铁律规则**：
自动 CQ 不再允许在奇偶窗口间乱飘。系统强制实行半小时轮换制：
- **0 ~ 29 分钟**：绝对钉死在 **偶数窗口 (Even)** 发送。
- **30 ~ 59 分钟**：绝对钉死在 **奇数窗口 (Odd)** 发送。
系统会在对应的历史底噪谱图中，找出一块 50Hz 的绝对寂静区，发出 `CQ MY_CALL MY_GRID`。

### 优先级 B：智能追击狩猎 (Auto Chase)
如果不满足 CQ 的条件，系统会遍历近期的解码记录，挑选目标：
1. **过滤黑名单**：剔除刚刚追击过（正在冷却中）的人，剔除今天已经通联过的相同国家/地区。
2. **贪婪排序算法**：按以下优先级对目标打分并排序：
   `超级远距离 (>4000km)  >  稀有 DXCC 地区  >  能保持我当前奇偶偏好的  >  带有 CQ 的消息  >  新的网格  >  信号信噪比 (SNR) 高的`
系统取出评分最高的第一名目标，生成回复，送入任务队列。

## 3. 回合制缠斗：智能跳频与无限陪跑

当系统成功咬住目标，双方开始互换信噪比（SNR）和确认信息时，系统拥有一套专门对付 QRM（信号干扰）的杀手锏。

### (1) Homing 交替跳频策略
你的每一次发射，系统都会运用 `repeat_count`（本阶段重试次数）来进行频率决策：
- **首发 (count: 0)**：使用本端计算的全新**寂静频率**。
- **复发 (count: 1)**：直接切换到**对方发消息时使用的频率**上（压脸输出）。
- **再发 (count: 2)**：切回**寂静频率**。
- **再发 (count: 3)**：切回**对方频率**。
以此类推，保证在遭遇局部频段干扰时，能用两种截然不同的策略击穿干扰。

### (2) “牛皮糖” 无限期陪跑机制
一般的软件在对方未回复 3-4 次后就会自动掐断放弃。我们的策略做到了极其聪明：
- 如果系统监听到对方**发来了重复的消息**（哪怕只是由于多路径传播导致 SNR 数字发生了 `+08` 到 `+04` 的变化），系统判定对方**还在尝试与你沟通**。
- 系统会立刻使用魔法重置：`repeat_count = repeat_count % 2`。
- **效果**：这保证了重试次数永远只会在 0 和 1 之间打转，**永远不会触碰 3-4 次的放弃上限**！只要他不放弃，你就会陪他发到天荒地老。同时，取模 `2` 完美继承了奇偶性，让你的“寂静/对方频率”交替跳频策略丝毫不受影响，如丝般顺滑。

## 4. 防弹设计与本地日志兜底

在自动通联的最底层，还有几套容错机制：
- **哈希呼号穿透**：当处理含有诸如 `R5AF/0` 等长呼号的非标准 FT8 消息时，WSJT-X 协议会将你的呼号压缩为一个 12 位的 Hash。因为你是本地发报（不过天空），本地缓存本不该有你的呼号。为此，`main.rs` 和 `parser.rs` 进行了特殊改造，在软件启动和每次发送 Echo 报文时，**强制将本地呼号及交互目标注入哈希解析池**，彻底杜绝 `<...>` 的乱码问题。
- **状态机死锁预防**：如果由于电脑网络问题或电台 IP 未就绪导致底层 UDP 发送失败，系统依然会正常累加计数器。这保证了哪怕硬件卡死，状态机依然会按时放弃死任务并回滚到空闲状态，随时准备下一轮追击，做到了逻辑层的绝对非阻塞。
- **Notion 及 PSK 日志上报**：每一次最终收到对方的 `73` 或 `RR73`，状态机才会判定一轮 QSO 完成，随后通过异步 Channel 触发 Notion API 写库与本地统计清理，干净利落地进入下一轮狩猎。
